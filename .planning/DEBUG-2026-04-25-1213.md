# Debug Report — 2026-04-25 12:13

**Symptom:** Vercel production build logged cron route `NEXT_PRERENDER_INTERRUPTED` errors while generating static pages.
**Mode:** general
**Tool calls used:** 8/10

## Investigation
- Diagnostic scans run: `rg request.headers app/api/cron`, local `npm run build`
- Files examined: all `app/api/cron/*/route.ts` handlers
- Patterns searched: `request.headers`, broad outer `catch`, `dynamic = 'force-dynamic'`

## Root Cause
`app/api/cron/*/route.ts` handlers read `request.headers` inside broad `try/catch` blocks. During build, Next throws a control-flow prerender bailout when it sees dynamic request data. The cron handlers caught that framework error and logged it as an application error.

## Fix Applied
- Files: all 9 cron route handlers under `app/api/cron/`
- Added `unstable_rethrow()` from `next/navigation` at the top of each outer catch block, before application logging.
- Preserved existing runtime error responses and cron auth checks.

## Verification
- `npx tsc --noEmit` — pass
- `npm run lint` — pass
- `npm run build` — pass, with no cron `NEXT_PRERENDER_INTERRUPTED` logs
- `npm test -- --runInBand` — pass, 655 tests

## Related Observations
- `dynamic = 'force-dynamic'` is incompatible with `nextConfig.cacheComponents` in this project, so it was not used.
