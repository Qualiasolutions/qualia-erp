**Goal**: Patch all IDOR vulnerabilities, harden auth and input validation, fix production observability gaps discovered in security audit
**Priority:** P0 — Must complete before public launch
**Plans:** 3 plans in 2 waves

Plans:

- [x] 25-01: Fix all IDOR vulnerabilities (client-requests, phase-comments, client-portal)
- [x] 25-02: Fix auth and invitation security + Zod validation on untrusted inputs
- [x] 25-03: Observability and hardening (removeConsole, health 503, .env.example, DB error sanitization, orphan rollback)

**Details:**

**Goal:** Patch all IDOR vulnerabilities, harden auth and input validation, fix production observability gaps discovered in security audit
**Depends on:** Phase 24 (Portal branding complete)
**Requirements:** SEC-01 through SEC-16 (all must-haves from security audit)
**Success Criteria:**

1. All IDOR vulnerabilities from audit patched with proper authorization checks
2. Invitation system uses opaque tokens preventing enumeration attacks
3. All portal user inputs validated through Zod schemas with proper error handling
4. Production builds preserve error/warn console output for observability
5. Health endpoint returns 503 for degraded states enabling reliable monitoring
6. Database errors sanitized before returning to clients
7. Orphaned Supabase auth users cleaned up on operation failures

## Milestone Summary

**Key Decisions:**

- Decision: Used existing canAccessProject from lib/portal-utils (Rationale: Avoid duplicate auth logic)
- Decision: Token-based invitation lookups vs UUID PKs (Rationale: Prevents enumeration attacks)
- Decision: removeConsole.exclude vs full removal (Rationale: Preserve error observability)
- Decision: Health 503 for degraded state (Rationale: Reliable monitoring signals)
- Decision: Orphan rollback best-effort pattern (Rationale: Don't mask original errors)

**Issues Resolved:**

- Fixed 4 IDOR vulnerabilities in client-requests, phase-comments, client-portal actions
- Resolved invitation enumeration attack vector through opaque token system
- Fixed production error visibility gap in build configuration
- Resolved health endpoint monitoring signal reliability
