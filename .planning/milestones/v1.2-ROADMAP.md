# Milestone v1.2: Premium Animations

**Status:** ✅ SHIPPED 2026-03-04
**Phases:** 10-11
**Total Plans:** 7

## Overview

Premium differentiator animations and final polish items. Added stagger animations, scroll-triggered reveals, spring physics, gesture-based drawers, activity feed pagination, useServerAction hook, and schedule utility consolidation. All building on v1.1's animation infrastructure and responsive foundations.

## Phases

### Phase 10: Differentiator Animations

**Goal**: Users experience premium, Apple-level animation polish throughout portal and trainee interfaces
**Depends on**: Phase 8 (v1.1 shipped)
**Plans**: 3 plans

Plans:

- [x] 10-01: Stagger animations on lists + scroll-triggered reveals on roadmap
- [x] 10-02: Spring physics on buttons and cards
- [x] 10-03: Gesture-based drawer interactions with Vaul

**Details:**

- Stagger: Index-based 50ms delay per item in virtualized task/inbox lists
- Scroll reveals: useInView hook with `once: true, margin: '-100px'` on phase cards
- Spring physics: CSS cubic-bezier for buttons, Framer Motion whileHover (stiffness 300, damping 20) for cards
- Gestures: Vaul library for mobile drawers (<768px), Dialog for desktop (>=768px)
- All animations respect prefers-reduced-motion

### Phase 11: Additional Polish

**Goal**: Portal and trainee system have production-grade data handling, formatting consistency, and reduced code duplication
**Depends on**: None (independent from Phase 10)
**Plans**: 4 plans

Plans:

- [x] 11-01: Activity feed cursor-based pagination
- [x] 11-02: Standardize date formatting across portal
- [x] 11-03: Create useServerAction hook to reduce form boilerplate
- [x] 11-04: Consolidate schedule utility files (3 → 2)

**Details:**

- Pagination: Cursor-based with ISO timestamps, 20 items/page, de-duplication by ID
- Date formatting: Centralized formatDate()/formatRelativeTime() from lib/utils
- useServerAction: Generic hook managing isPending/error/isSuccess with optimistic update support
- Schedule consolidation: Merged schedule-shared.ts into schedule-utils.ts (353 lines)

## Milestone Summary

**Key Decisions:**

- SPRING-01: CSS spring curves for buttons (preserves Slot.Root pattern)
- SPRING-02: Framer Motion whileHover for cards (stiffness 300, damping 20)
- DRAWER-01: Vaul over custom implementation (built-in gesture physics, accessibility)
- DRAWER-02: FormContent extraction for Drawer/Dialog code sharing
- PAGINATION-01: Cursor-based pagination over offset-based (prevents duplicates)
- HOOK-01: useServerAction hook pattern for standardized form state management
- SCHEDULE-CONSOLIDATE-01: Merge schedule-shared.ts into schedule-utils.ts

**Issues Resolved:**

- Activity feed loading all items at once (now paginated)
- Inconsistent date formatting across portal components
- Form submission boilerplate duplicated across 30+ components
- Schedule logic split across 3 files

**Issues Deferred:**

- useServerAction adoption across remaining ~28 components
- Real device testing for gesture drawers (iOS Safari, Android Chrome)

**Technical Debt Incurred:**

- schedule-utils.ts marked 'use client' for useTimezone hook (scoring functions don't need client-side)
