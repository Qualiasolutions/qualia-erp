## v1 Requirements

Requirements for v1.4 Admin Portal Onboarding milestone.

### Project Import

- [x] **IMPORT-01**: Admin can view ERP projects that are not yet portal-enabled — filter tab "Not Enabled", 3-tier badge system
- [x] **IMPORT-02**: Admin can select multiple ERP projects for bulk import to portal — Set-based multi-select, bulk toolbar
- [x] **IMPORT-03**: Admin can preview what client will see before enabling portal access — RoadmapPreviewModal with phase timeline
- [x] **IMPORT-04**: Admin can configure project-specific portal settings during import — PortalSettingsModal with welcome message + visibility toggles
- [x] **IMPORT-05**: Admin can save portal configuration and mark projects ready for client invitation — metadata.portal_settings persists, badge updates to "Portal Ready"

### Client Invitation

- [x] **INVITE-01**: Admin can enter client email address for a portal-enabled project — SendInvitationModal with email input
- [x] **INVITE-02**: Admin can send invitation email with project details to client — sendClientInvitation() via Resend
- [x] **INVITE-03**: Admin can resend invitation if client hasn't responded — InvitationHistoryModal resend button
- [x] **INVITE-04**: Admin can view invitation status (sent, delivered, opened, account created) — 4-state color-coded badges
- [x] **INVITE-05**: System tracks invitation history and status changes — client_invitations table, timeline modal

### Client Onboarding

- [x] **ONBOARD-01**: Client receives well-designed invitation email with clear call-to-action — HTML email with gradient header, CTA button
- [x] **ONBOARD-02**: Client can click invitation link to access account creation page — /auth/signup?token={TOKEN}
- [x] **ONBOARD-03**: Client can create account with email, password, and basic profile — SignupForm with pre-filled email, password validation
- [x] **ONBOARD-04**: Client is automatically logged in after account creation — signUp() creates session automatically
- [x] **ONBOARD-05**: Client has immediate access to their specific project after signup — Auto-redirect to /portal/[projectId]
- [x] **ONBOARD-06**: Client can access project roadmap, files, and comments immediately — Portal content renders all features

## Out of Scope

| Feature                       | Reason                                             |
| ----------------------------- | -------------------------------------------------- |
| SSO/OAuth integration         | Email/password sufficient for v1, adds complexity  |
| Client self-service import    | Admin-controlled onboarding ensures quality        |
| Real-time invitation tracking | Email delivery status sufficient, avoid complexity |
| White-label customization     | Single-tenant system, standardized branding        |
