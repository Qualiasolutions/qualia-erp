## Milestone Summary

**Key Decisions:**

- Soft delete pattern for project assignments — preserves complete audit trail
- Partial unique index for active assignments only — prevents duplicates while allowing history
- Admin-only assignment mutations — centralized control over employee assignments
- Database view for integration mappings — portal_project_mappings avoids N+1 queries
- Rich status objects over booleans — structured integration status for better UI feedback
- 45s SWR refresh interval — balances real-time feel with API efficiency
- Path revalidation on all mutations — simpler than SWR cache invalidation
- Notification routing via project_assignments — leverages Phase 12 foundation
- Activity log with is_client_visible flag — enables unified ERP/portal timeline
- Hybrid server-client pattern for portal pages — auth on server, data fetching on client for real-time sync
- Subtle validating indicators over full overlays — better UX during background refresh
- Default all notification types enabled with 'both' delivery — ensures users don't miss critical notifications
- shouldSendEmail fails safe — returns true on error, better to send extra email than miss notification
- Email notifications to assigned employees only — leverages Phase 12 assignments, reduces noise
- Silent failure pattern for emails — delivery failures don't block user actions
- Qualia teal gradient branding in emails — #00A4AC to #008B92 for brand consistency
- Fire-and-forget notification pattern — email sending doesn't block HTTP responses
- Self-fetching settings components — consistent with LearnModeSettings, avoids prop drilling
- Leveraged existing project_files table for features gallery — is_client_visible flag enables selective sharing
- Phase badges in gallery overlays and lightbox — provides context for feature delivery timeline
- Hybrid server-client pattern for portal dashboard — thin server auth layer, SWR client data fetching for real-time sync
- 45s auto-refresh with subtle indicators — balances real-time feel with API efficiency, non-intrusive UX
