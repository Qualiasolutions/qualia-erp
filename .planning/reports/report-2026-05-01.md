## What Was Done

- **Codified four invoice patterns as a reusable template system** on `/admin?tab=finance`. The "New invoice from template" dialog replaces the manual workflow of looking up Zoho contact IDs, copying line items from prior invoices, building terms by hand, and saving cover-email drafts one tool call at a time. Templates: `monthly_retainer`, `simple_service`, `project_deposit`, `project_balance`, plus two terms templates (`generic`, `sakani_pda`).
- **Extended the Zoho Books integration** (`lib/integrations/zoho.ts`) to support `customer_id`, `tax_id` per line, `terms`, `reference_number` and draft mode on `createZohoInvoice`. Added `createZohoMailDraft` so the system can save a Zoho Mail cover-email draft alongside each invoice (Verdana 10pt, "Dear X" tone matching Fawzi's existing send pattern).
- **Backfilled the client → Zoho contact mapping** via migration `20260501030000_finance_template_system.sql` (applied to live DB). 13 ERP clients are now linked to their Zoho `contact_id` with a `default_vat_treatment` (Cyprus 19% / non-EU 0%) — these are the only clients invoiceable through the new dialog.
- **Exposed the system through 5 new MCP tools** at `/api/mcp/mcp` (server v1.1.0): `list_invoice_templates`, `list_billable_clients`, `list_invoices`, `create_invoice_draft`, `create_invoice_cover_email_draft`. Lets the qualia-framework drive billing from any project (e.g. `/qualia-quick "draft May retainer invoices"`).
- **Refactored to a workspace-scoped core** (`lib/invoice-generation-core.ts`) so both the server-action wrappers (used by the dialog) and the MCP route can call the same orchestration with their own auth contexts (session vs token-derived).
- **Wrote handoff runbook** (`docs/finance-runbook.md`) covering once-per-client setup, single-invoice workflow, bulk monthly retainer batch, and common error remediation. Designed so a non-engineer can run monthly billing without touching Zoho directly.
- **Tonight earlier**: hand-stitched 4 actual invoices via Zoho MCP (Underdog INV-10016 €780, Maison Maud INV-10017 €500, Armenius INV-10018 €357, EventMaster INV-10019 €1,071) plus 1 Zoho Mail draft for Tasos. That manual workflow is what motivated this build.
