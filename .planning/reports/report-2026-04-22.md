- **Auto-fix scope was split from Phase 23 scope** so they could run in parallel without merge conflicts. Hard exclusion list enforced: builder could not touch any file Phase 23 owned. Worked cleanly, no conflicts.
- **Phase 23 H2 picked Option A** (rename `'client_file_uploaded'` → `'file_uploaded'` + `actionData.is_client_upload: true`) over adding a new enum value + migration. Grep confirmed zero readers depend on the old string.
- **Phase 23 H3 picked Option A** (new RLS policy, workspace-scoped) over `createAdminClient()` bypass. RLS is the proper long-term answer; the migration is a single `CREATE POLICY`. Policy checks both admin role AND workspace membership in both USING and WITH CHECK.
- **Phase 24 T2 billing reverted to static import** — `app/(portal)/billing/page.tsx` is an async Server Component and `ssr: false` isn't permitted in Server Components per Next.js 16 App Router. The dialog has `'use client'` at its top, so Next.js already code-splits it automatically — the dynamic wrapper added no bundle benefit and broke the build. Other 3 files kept dynamic imports (all are Client Components).
- **Phase 24 T4 phases+tasks parallel, resources sequential** — `phase_resources` table has no `project_id` column, so the resources query must wait for phase_ids. Two roundtrips instead of three still saves ~100-200ms per project-detail open.
