    **Shared test utilities (`__tests__/actions/test-utils.ts`):**

    Create a Supabase mock factory that action tests can reuse. The factory must:

    1. Mock `@/lib/supabase/server` — the `createClient` function must return a mock Supabase client
    2. The mock client needs chainable query builder: `.from().select().eq().order()` etc.
    3. Provide helpers to configure mock responses: `mockSupabaseResponse(data, error)` that sets what the chain returns
    4. Mock `auth.getUser()` to return a configurable user (default: `{ id: 'test-user-id', email: 'test@example.com' }`)
    5. Export `setupActionMocks()` that returns `{ supabase, mockUser }` for each test

    Pattern for chainable mock:
    ```ts
    const createChainableMock = (resolvedValue: { data: any; error: any }) => {
      const chain: any = {};
      const methods = ['select', 'insert', 'update', 'delete', 'eq', 'neq', 'in', 'is',
        'order', 'limit', 'single', 'maybeSingle', 'range', 'ilike', 'or', 'not',
        'gte', 'lte', 'gt', 'lt', 'match', 'contains', 'textSearch'];
      methods.forEach(m => { chain[m] = jest.fn().mockReturnValue(chain); });
      // Terminal methods resolve the promise
      chain.then = (resolve: any) => resolve(resolvedValue);
      chain.select = jest.fn().mockReturnValue(chain);
      return chain;
    };
    ```

    Also mock `revalidatePath` from `next/cache` since actions call it after mutations.

    **Inbox tests (`__tests__/actions/inbox.test.ts`):**

    Read `app/actions/inbox.ts` carefully. Test every exported function. Key functions to test:

    - `getInboxTasks()` — success path (returns tasks), empty result, error from Supabase
    - `createTask()` — valid input creates task, missing required fields returns error, Supabase error returns ActionResult with error
    - `updateTask()` — valid update succeeds, non-existent task returns error, auth check (non-owner can't update others' tasks if applicable)
    - `deleteTask()` — success, authorization check, Supabase error
    - `reorderTasks()` — valid reorder, empty array
    - `updateTaskStatus()` — status transitions
    - `toggleInbox()` — toggle show_in_inbox flag

    For each function, test:
    1. Happy path with valid input
    2. Error path (Supabase returns error)
    3. Auth path (no user / unauthorized)
    4. Validation path (invalid input rejected by Zod)

    Use `jest.mock('@/lib/supabase/server')` at the top. Configure mock responses per test using the shared utilities.

    Important: These are server actions with `'use server'` directive. Jest can still import and test them — the directive is ignored in test environment. But you MUST mock `createClient` since there's no real Supabase in tests.
