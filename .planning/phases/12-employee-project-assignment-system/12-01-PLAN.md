  <name>Task 1: Create project_assignments table with RLS policies</name>
  <files>supabase/migrations/[timestamp]_create_project_assignments.sql</files>
  <action>
Create Supabase migration for project_assignments table using Supabase MCP tools.

**Table schema:**

- id: uuid primary key default gen_random_uuid()
- project_id: uuid not null references projects(id) on delete cascade
- employee_id: uuid not null references profiles(id) on delete cascade
- assigned_by: uuid not null references profiles(id) - admin who made assignment
- assigned_at: timestamptz not null default now()
- removed_at: timestamptz null - when assignment ended (null = currently active)
- removed_by: uuid null references profiles(id) - who removed assignment
- workspace_id: uuid not null references workspaces(id) on delete cascade
- notes: text null - optional assignment context

**Indexes:**

- index on project_id for quick project lookups
- index on employee_id for quick employee lookups
- index on workspace_id for RLS filtering
- composite index on (project_id, employee_id, removed_at) for active assignment checks

**Constraints:**

- unique constraint on (project_id, employee_id) where removed_at is null - prevents duplicate active assignments

**RLS policies:**

1. Enable RLS on table
2. SELECT policy: workspace members can view assignments in their workspace
3. INSERT policy: admin role only
4. UPDATE policy: admin role only (for setting removed_at/removed_by)
5. DELETE policy: admin role only (hard delete for mistakes)

Use Supabase MCP to create migration, apply with `npx supabase db push`, then regenerate types with `npx supabase gen types typescript`.

Create server actions file following existing patterns from app/actions/projects.ts and app/actions/shared.ts.

**Actions to implement:**

1. **assignEmployeeToProject(formData: FormData): Promise&lt;ActionResult&gt;**
   - Validate: project_id, employee_id, notes (optional)
   - Auth check: isUserAdmin() required
   - Workspace check: verify both project and employee belong to same workspace
   - Duplicate check: ensure no active assignment exists (removed_at is null)
   - Insert: project_id, employee_id, assigned_by (current user), workspace_id, notes
   - Activity log: create activity for assignment
   - Revalidate: revalidatePath(`/projects/${project_id}`) and revalidatePath('/admin/assignments')
   - Return: ActionResult with assignment data

2. **reassignEmployee(formData: FormData): Promise&lt;ActionResult&gt;**
   - Validate: assignment_id, new_project_id, notes (optional)
   - Auth check: isUserAdmin() required
   - Transaction: remove old assignment (set removed_at, removed_by) + create new assignment atomically
   - Revalidate paths for both old and new projects
   - Return: ActionResult with new assignment

3. **removeAssignment(assignmentId: string): Promise&lt;ActionResult&gt;**
   - Auth check: isUserAdmin() required
   - Update: set removed_at = now(), removed_by = current user
   - Activity log: create activity for removal
   - Revalidate: affected project and admin pages
   - Return: ActionResult

4. **getProjectAssignments(projectId: string): Promise&lt;ActionResult&gt;**
   - Fetch all assignments for project (active only by default)
   - Include employee profile data via FK join: full_name, email, avatar_url
   - Normalize FK response using normalizeFKResponse helper
   - Return: ActionResult with assignments array

5. **getEmployeeAssignments(employeeId: string): Promise&lt;ActionResult&gt;**
   - Fetch all assignments for employee (active only by default)
   - Include project data via FK join: name, status, client info
   - Normalize FK response
   - Return: ActionResult with assignments array

6. **getAssignmentHistory(projectId?: string, employeeId?: string): Promise&lt;ActionResult&gt;**
   - Fetch ALL assignments including removed ones for audit trail
   - Filter by project_id OR employee_id if provided
   - Include both employee and project data
   - Order by assigned_at desc
   - Return: ActionResult with full history

- Import helpers from ./shared (isUserAdmin, createActivity, normalizeFKResponse)
