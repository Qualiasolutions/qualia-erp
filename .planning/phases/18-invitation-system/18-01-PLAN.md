  <name>Task 1: Create client_invitations database schema</name>
  <files>supabase/migrations/[timestamp]_client_invitations.sql</files>
  <action>
Create Supabase migration for client invitation tracking:

**Table: client_invitations**

- id: uuid PRIMARY KEY DEFAULT gen_random_uuid()
- project_id: uuid NOT NULL REFERENCES projects(id) ON DELETE CASCADE
- email: text NOT NULL (client email address)
- invitation_token: text NOT NULL UNIQUE (cryptographically secure, used in signup link)
- status: invitation_status NOT NULL DEFAULT 'sent'
- invited_by: uuid NOT NULL REFERENCES auth.users(id)
- invited_at: timestamptz DEFAULT now()
- resent_at: timestamptz (last resend timestamp)
- resent_count: int DEFAULT 0
- opened_at: timestamptz (when client clicked link - **populated by Phase 19 signup flow**)
- account_created_at: timestamptz (when client completed signup - **populated by Phase 19 signup flow**)
- metadata: jsonb DEFAULT '{}'::jsonb (for future extensibility)
- CONSTRAINT unique_project_email UNIQUE(project_id, email)

**Enum: invitation_status**

- 'sent' (initial state after createInvitation)
- 'resent' (after resendInvitation called)
- 'opened' (client clicked invitation link - **Phase 19 sets this when /auth/signup?token=X is loaded**)
- 'accepted' (client created account - **Phase 19 sets this when client completes registration**)
- 'expired' (future feature - v2)

**Indexes:**

- idx_invitations_project ON client_invitations(project_id, invited_at DESC)
- idx_invitations_token ON client_invitations(invitation_token) WHERE status NOT IN ('accepted', 'expired')
- idx_invitations_status ON client_invitations(status)

**RLS Policies:**

- "Admin full access to invitations" FOR ALL USING (is_admin())
- "Managers view invitations for their projects" FOR SELECT USING (EXISTS (SELECT 1 FROM projects p WHERE p.id = client_invitations.project_id AND p.lead_id = auth.uid()))

Enable RLS on client_invitations table.

**Why this structure:**

- invitation_token is cryptographically secure (will use crypto.randomUUID() in server action)
- UNIQUE constraint on (project_id, email) prevents duplicate invitations per project
- status transitions: sent → resent (optional) → opened → accepted
- resent_count tracks number of resends for analytics
- metadata JSONB for future features without schema changes
- Indexes optimize common queries: project history, token lookup, status filtering

**Phase 18 vs Phase 19 scope:**

- Phase 18 creates opened_at and account_created_at columns in schema but does NOT populate them
- Phase 19 implements /auth/signup?token=X route which sets opened_at and account_created_at
- This plan establishes infrastructure; Phase 19 implements automatic status transitions

**1. Add validation schema (lib/validation.ts):**

```typescript
export const invitationSchema = z.object({
  projectId: z.string().uuid(),
  email: z.string().email().trim().toLowerCase(),
});

export type InvitationInput = z.infer<typeof invitationSchema>;
```

**2. Create server actions (app/actions/client-invitations.ts):**

Import pattern: createClient from @/lib/supabase/server, ActionResult from ./shared, invitationSchema from @/lib/validation.

**createInvitation(input: InvitationInput): Promise<ActionResult>**

- Validate input with invitationSchema
- Get current user and verify admin/manager role
- Check if invitation already exists for this project+email (return existing if status is 'sent' or 'resent')
- Generate secure token: crypto.randomUUID() (built-in Node.js, cryptographically secure)
- Insert into client_invitations table with status='sent'
- Return { success: true, data: { invitationId, token, email } }
- Error handling: return { success: false, error: message }

**resendInvitation(invitationId: string): Promise<ActionResult>**

- Verify admin/manager role
- Update invitation: status='resent', resent_at=now(), resent_count=resent_count+1
- Return { success: true, data: { invitationId, email } }
- Error: invitation not found or already accepted

**getInvitationHistory(projectId: string): Promise<ActionResult>**

- Verify admin/manager role for project
- Query client_invitations WHERE project_id = projectId ORDER BY invited_at DESC
- Join with profiles table to get invited_by user name
- Return { success: true, data: invitations[] }

**getProjectInvitationStatus(projectId: string): Promise<{ hasInvitation: boolean; status: string | null; email: string | null }>**

- Query most recent invitation for project
- Return status info for UI badges
- Used by Plan 02 for status display

**Authorization pattern:**

- Use existing is_admin() check or project lead verification
- Match Phase 17 pattern: getAdminStatus() or project ownership query

**Error handling:**

- All functions return ActionResult with success/error structure
- Database errors caught and logged, user-friendly messages returned
- Duplicate invitation returns existing record (idempotent)
