    **Vercel webhook filter injection (app/api/webhooks/vercel/route.ts line 90):**

    The current code interpolates external input directly into the `.or()` filter:
    ```typescript
    .or(`vercel_project_id.eq.${project.id},name.ilike.%${project.name}%`)
    ```

    `project.id` and `project.name` come from the Vercel webhook payload (external input). A malicious payload could inject PostgREST filter operators.

    Fix: Replace the single `.or()` call with two separate queries, or sanitize the inputs. The cleanest approach is to split into two sequential lookups:

    ```typescript
    // First try exact match on vercel_project_id
    let { data: ourProject } = await supabase
      .from('projects')
      .select('id, name')
      .eq('vercel_project_id', project.id)
      .limit(1)
      .single();

    // Fallback: fuzzy match on name (only if ID match failed)
    if (!ourProject) {
      const { data: nameMatch } = await supabase
        .from('projects')
        .select('id, name')
        .ilike('name', `%${project.name.replace(/[%_]/g, '')}%`)
        .limit(1)
        .single();
      ourProject = nameMatch;
    }
    ```

    Note: Also strip `%` and `_` from `project.name` since those are SQL LIKE wildcards. This prevents a crafted project name from matching unintended rows.

    **Timing-safe API key comparison (all 3 claude routes):**

    In each of the 3 `/api/claude/*` routes, replace the simple `===` comparison:
    ```typescript
    // OLD
    if (!expectedKey || apiKey !== expectedKey) {
    ```

    With a timing-safe comparison using `crypto.timingSafeEqual()`:
    ```typescript
    import crypto from 'crypto';

    function safeCompare(a: string | null, b: string | undefined): boolean {
      if (!a || !b) return false;
      if (a.length !== b.length) return false;
      return crypto.timingSafeEqual(Buffer.from(a), Buffer.from(b));
    }

    // Then in the handler:
    if (!expectedKey || !safeCompare(apiKey, expectedKey)) {
    ```

    To avoid duplication, create a shared helper. Add a `lib/auth-utils.ts` file with the `safeCompare` function, then import it in all 3 routes. Or if you prefer to keep it simple, inline the helper in each file (3 files, small function — either approach is fine).

    The files to update:
    - `app/api/claude/project-status/route.ts`
    - `app/api/claude/session-feed/route.ts`
    - `app/api/claude/session-log/route.ts`
