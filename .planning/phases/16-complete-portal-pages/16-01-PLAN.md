**2. updateNotificationPreferences(data)**

- Validates using notificationPreferencesSchema
- Gets auth user
- Updates notification preferences (use JSON column on profiles table: notification_preferences)
- Structure: { email_enabled: boolean, types: string[], frequency: 'instant' | 'daily' | 'weekly' }
- Returns ActionResult

Add to lib/validation.ts:

```typescript
export const clientProfileSchema = z.object({
  full_name: z.string().min(2, 'Name must be at least 2 characters').max(100).optional(),
  company: z.string().max(100).optional(),
});

export const notificationPreferencesSchema = z.object({
  email_enabled: z.boolean(),
  types: z.array(
    z.enum(['project_updates', 'phase_completions', 'new_messages', 'invoice_updates'])
  ),
  frequency: z.enum(['instant', 'daily', 'weekly']),
});

export type ClientProfileInput = z.infer<typeof clientProfileSchema>;
export type NotificationPreferencesInput = z.infer<typeof notificationPreferencesSchema>;
```

Follow existing server action patterns in client-portal.ts (auth check, error handling, return ActionResult).
