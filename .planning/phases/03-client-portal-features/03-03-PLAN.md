<task type="auto">
  <name>Create phase comments server actions</name>
  <files>app/actions/phase-comments.ts</files>
  <action>
Create server actions for phase-level comments:

1. **createPhaseComment(data: { projectId: string; phaseName: string; commentText: string; isInternal?: boolean })** → ActionResult<PhaseComment>
   - Get current user ID from session
   - Validate inputs (commentText not empty, max 2000 chars)
   - Insert into phase_comments table:
     - project_id
     - phase_name
     - commented_by (current user ID)
     - comment_text
     - is_internal (default: false for clients, can be true for admin/employee)
     - created_at (auto)
   - If user is client: force is_internal = false (clients can't create internal comments)
   - revalidatePath(`/portal/${projectId}`)
   - revalidatePath(`/projects/${projectId}/roadmap`)
   - Return created comment with user profile joined

2. **getPhaseComments(projectId: string, phaseName: string, includeInternal = false)** → ActionResult<PhaseComment[]>
   - Query phase_comments WHERE project_id AND phase_name
   - If includeInternal = false: add WHERE is_internal = false
   - Join with profiles for commented_by user info (name, avatar)
   - ORDER BY created_at ASC (chronological)
   - Use normalizeFKResponse for profile join
   - Return array of comments with user data

3. **deletePhaseComment(commentId: string, projectId: string)** → ActionResult
   - Validate: user is admin OR user is comment author
   - Query comment to verify ownership
   - Delete from phase_comments WHERE id = commentId
   - revalidatePath(`/portal/${projectId}`)
   - revalidatePath(`/projects/${projectId}/roadmap`)
   - Return success

4. **getProjectCommentsCount(projectId: string, clientVisibleOnly = false)** → ActionResult<number>
   - Count comments WHERE project_id = projectId
   - If clientVisibleOnly: add WHERE is_internal = false
   - Return count
   - Used for showing comment count badges

Follow existing action patterns with ActionResult, error handling, and revalidation.

Note: phase_comments table already exists with all required columns (id, project_id, phase_name, task_key, commented_by, comment_text, is_internal, created_at).
</action>
