  <name>Task 1: Add getPortalClientManagement server action with last login data</name>
  <files>app/actions/client-portal.ts</files>
  <action>
    Add `getPortalClientManagement(): Promise<ActionResult>` exported function.

    Implementation:
    1. Auth + admin check (same pattern as `getPortalAdminData`)
    2. Query `profiles` for all role='client': `id, full_name, email, created_at`
    3. Query `client_projects` with project join to get all assignments: `client_id, project_id, project:projects!project_id(id, name)`
    4. Get last login data using admin client:
       ```typescript
       const adminClient = createAdminClient();
       const { data: authData } = await adminClient.auth.admin.listUsers({ perPage: 1000 });
       ```
       Build a map: `Map<email, last_sign_in_at>` from `authData.users`. Match by email (normalize to lowercase).
    5. Merge: for each client profile, attach:
       - `lastSignIn: string | null` — from the auth map by email
       - `isActive: boolean` — true if lastSignIn exists and is within 30 days of now
       - `projects: Array<{ id: string; name: string }>` — from assignments
    6. Return: `{ success: true, data: { clients: MergedClient[], totalActive: N, totalInactive: N } }`

    Type for MergedClient:
    ```typescript
    {
      id: string;
      full_name: string | null;
      email: string | null;
      created_at: string;
      lastSignIn: string | null;
      isActive: boolean;
      projects: Array<{ id: string; name: string }>;
    }
    ```

    Note: `adminClient.auth.admin.listUsers` paginates at 1000 by default. For a small agency this is fine. Add a comment noting this limit.

  </action>
