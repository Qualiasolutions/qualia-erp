  <action>
Add a new exported async function `setupPortalForClient(clientId: string, projectIds: string[]): Promise<ActionResult>` to `app/actions/client-portal.ts`.

The function must:

1. Auth check — `supabase.auth.getUser()` + `isUserManagerOrAbove`. Return error if fails.
2. Look up the CRM client: `supabase.from('clients').select('id, name, contacts').eq('id', clientId).single()`. The `contacts` column is a JSONB array — extract the first contact's email (`contacts[0]?.email`). If no email found, return `{ success: false, error: 'CRM client has no email on file' }`.
3. Check for an existing portal profile with that email: `supabase.from('profiles').select('id, role').eq('email', normalizedEmail).single()`.
   - If found with role admin/manager/employee: return error "This email belongs to a team member".
   - If found with role client: skip creation, use existing profile id.
4. If no profile: create auth user via `adminClient.auth.admin.createUser` with `email: normalizedEmail`, `email_confirm: true`, `user_metadata: { role: 'client', full_name: client.name }`, and `password: \`Qualia-\${crypto.randomBytes(12).toString('base64url')}\``(import`crypto`from`'node:crypto'` at top of file — check if already imported first).
5. Upsert profile via adminClient with `id, email, full_name: client.name, role: 'client'`.
6. For each projectId in `projectIds`: check if `client_projects` link already exists, skip if so, otherwise insert `{ client_id, project_id, invited_by: user.id, invited_at: new Date().toISOString() }`. Collect any insert errors but do not fail the whole operation for duplicates.
7. If auth user was newly created but ALL project inserts failed: call `adminClient.auth.admin.deleteUser(newUserId)` to roll back the orphan, then return error.
8. `revalidatePath('/portal')`, `revalidatePath('/clients')`, `revalidatePath('/projects')`.
9. Return `{ success: true, data: { userId, email: normalizedEmail, name: client.name, tempPassword (if newly created), alreadyExisted: boolean, projectsLinked: number } }`.

Keep existing `setupClientForProject` and `inviteClientByEmail` functions intact — do not remove them.
