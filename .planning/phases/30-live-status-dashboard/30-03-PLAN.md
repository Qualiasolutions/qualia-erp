Create `components/today-dashboard/session-history-panel.tsx` as a 'use client' component.

**Props:**

```typescript
interface SessionHistoryPanelProps {
  workspaceId: string;
  profileId: string;
  profileName: string;
  onClose: () => void;
}
```

**State:**

- `selectedDate`: string (YYYY-MM-DD), defaults to today
- `sessions`: fetched via `getSessionsAdmin(workspaceId, { profileId, date: selectedDate })`
- Use SWR or direct fetch with useEffect -- prefer using the existing `useSessionsAdmin` pattern if it exists in swr.ts, otherwise use a local `useSWR` call with `cacheKeys.sessionsAdmin(workspaceId, profileId, selectedDate)` as the key.

**Layout:**

- Header: employee name + close button (X icon)
- Date picker: use shadcn `Button` with left/right arrows to navigate dates + a display of the current date formatted as "Wed, Mar 27". Include a "Today" button to jump back.
- Session list: For each session, render a row showing:
  - Time range: `format(started_at, 'HH:mm')` - `format(ended_at, 'HH:mm')` or "ongoing" if ended_at is null
  - Project name (or "No project" in muted text)
  - Duration: computed from started_at/ended_at, formatted as "Xh Ym". For ongoing sessions, show ticking duration (reuse DurationTicker pattern from 30-02).
  - Summary text (if exists), shown below the row in smaller muted text
- Total hours for the day at the bottom: sum of all session durations
- Empty state: "No sessions on this date"
- Loading state: skeleton rows

**Styling:**

- Same tinted neutral palette as rest of dashboard
- Compact, data-dense rows
- Use `date-fns` for all date formatting (format, formatDistanceToNow, addDays, subDays)
- Session with no ended_at: show green "Active" badge
