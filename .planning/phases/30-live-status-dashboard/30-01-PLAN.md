<task type="auto">
  <name>Task 1: Add getTeamStatus server action</name>
  <files>app/actions/work-sessions.ts</files>
  <action>
Add a new exported type and server action to `app/actions/work-sessions.ts`:

1. Define `TeamMemberStatus` interface:

```typescript
export interface TeamMemberStatus {
  profileId: string;
  fullName: string | null;
  avatarUrl: string | null;
  status: 'online' | 'offline';
  // When online:
  projectName: string | null;
  sessionStartedAt: string | null;
  // When offline:
  lastSessionEndedAt: string | null;
}
```

2. Create `getTeamStatus(workspaceId: string): Promise<TeamMemberStatus[]>` action:
   - Auth check + `isUserAdmin()` guard (same pattern as `getSessionsAdmin`)
   - Fetch all profiles in workspace: `supabase.from('profiles').select('id, full_name, avatar_url').eq('workspace_id', workspaceId)`
   - Fetch all OPEN sessions (ended_at IS NULL) for the workspace: `supabase.from('work_sessions').select('profile_id, started_at, project:projects!work_sessions_project_id_fkey (id, name)').eq('workspace_id', workspaceId).is('ended_at', null)`
   - For offline employees, fetch their most recent closed session: use a single query with `.order('ended_at', { ascending: false })` and group in JS by profile_id (take first per profile)
   - Normalize FK arrays (project) using the same pattern as existing actions
   - Map profiles to TeamMemberStatus: if profile has an open session -> status 'online' with project and startedAt; otherwise -> status 'offline' with lastSessionEndedAt
   - Return sorted: online members first, then offline, alphabetically within each group

Keep the query count minimal (3 queries max: profiles, open sessions, recent closed sessions).
</action>
