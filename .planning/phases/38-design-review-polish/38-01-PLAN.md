  <files>.planning/phases/38-design-review-polish/38-01-AUDIT.md</files>
  <action>
Run a `/critique` design audit on the following pages by reading their source code and checking against the Impeccable v4.0 spec defined in globals.css and CLAUDE.md:

**Pages to audit:**

1. **Dashboard** (`app/today-page.tsx` + `components/today-dashboard/`) — check visual hierarchy, spacing consistency, surface layers, tinted neutrals
2. **Projects list** (`app/projects/page.tsx`) — check for pure grays, consistent card surfaces, proper elevation usage
3. **Project detail** (`app/projects/[id]/page.tsx`) — check surface layering, spacing, visual density
4. **Schedule** (`app/schedule/page.tsx`) — check design language consistency with dashboard

**For each page, check these Impeccable v4.0 criteria:**

- Tinted neutrals: No `gray-*`, `slate-*`, `zinc-*`, `neutral-*` Tailwind classes (should use CSS variables or `qualia-*` or `muted`/`secondary`/`border` semantic tokens)
- No pure black (`text-black`, `#000000`) or pure white (`bg-white`, `#ffffff`) — should be tinted
- Surface layers: Uses `bg-card`, `bg-background`, `hsl(var(--surface-*))` for depth
- Elevation: Uses `shadow-elevation-*` or `shadow-depth-*` system, not arbitrary `shadow-sm/md/lg`
- Spacing: Consistent use of spacing scale, no random pixel values
- Typography: Uses Geist font (via `font-sans`), proper size hierarchy
- Motion: Uses `ease-out-quart`/`ease-out-expo`/`premium` timing, no `ease-in-out` or `linear`
- No anti-patterns: No cards-in-cards, no gray-on-color text, no uniform spacing monotony

**For each violation found, record:**

- Page and component file path
- Tailwind class or CSS value causing the violation
- Which Impeccable v4.0 principle it violates
- Severity: CRITICAL (breaks design language), HIGH (visible inconsistency), MEDIUM (minor), LOW (nitpick)
- Suggested fix

Write initial findings to `38-01-AUDIT.md` with sections per page.
</action>
