Add two new server actions to app/actions/client-invitations.ts for signup flow:

**1. getInvitationByToken(token: string)**

- Query client_invitations table WHERE invitation_token = token
- Join with projects table to get project name, workspace_id
- Extract portal_settings.welcomeMessage from project.metadata JSONB
- Return: { invitation: { id, email, project_id, project_name, welcome_message, workspace_id }, error?: string }
- Error cases: token not found, invitation already accepted, invitation expired
- Use regular supabase client (read operation, RLS allows public read by token)

**2. markInvitationOpened(invitationId: string)**

- Update client_invitations SET status='opened', opened_at=now() WHERE id=invitationId
- Only update if status is 'sent' or 'resent' (don't overwrite 'accepted')
- Use createAdminClient() since this is called before user auth exists
- Return ActionResult { success, error? }

**3. markInvitationAccepted(invitationId: string)**

- Update client_invitations SET status='accepted', account_created_at=now() WHERE id=invitationId
- Use createAdminClient() since called during signup before session established
- Return ActionResult { success, error? }

**Implementation notes:**

- Import createAdminClient from lib/supabase/server
- Use normalizeFKResponse for project FK array
- Handle database errors gracefully with try/catch
- Add 'use server' directive

  <files>app/actions/auth.ts</files>
  <action>
Add signupWithInvitationAction to app/actions/auth.ts:

**Function signature:**

```typescript
export async function signupWithInvitationAction(
  _prevState: { success: boolean; error: string | null; projectId?: string },
  formData: FormData
): Promise<{ success: boolean; error: string | null; projectId?: string }>;
```

**Implementation flow:**

1. Extract formData fields: email, password, fullName, invitationToken
2. Validate required fields (email, password, fullName, invitationToken)
3. Call getInvitationByToken(invitationToken) to validate and get invitation details
4. If invalid: return { success: false, error: 'Invalid or expired invitation' }
5. Create Supabase auth user via regular client: `supabase.auth.signUp({ email, password })`
6. If signUp fails: return error
7. Get new user ID from signUp response: `data.user.id`
8. Use createAdminClient() to create profile and client_projects (RLS won't allow new user to write):
   - Insert into profiles: { id: userId, email, full_name: fullName, role: 'client' }
   - Insert into client_projects: { client_id: userId, project_id: invitation.project_id, workspace_id: invitation.workspace_id, access_level: 'comment', invited_by: get from invitation }
9. Call markInvitationAccepted(invitation.id)
10. Return { success: true, error: null, projectId: invitation.project_id }

**Security notes:**

- Use createAdminClient() ONLY for profile and client_projects inserts (user can't write to these yet)
- Regular client for signUp (uses anon key, proper auth flow)
- Verify invitation is valid BEFORE creating auth user (don't create orphan accounts)

**Error handling:**

- Wrap entire flow in try/catch
- If profile/client_projects insert fails after signUp, auth user exists but can't login (acceptable for v1, will show error)
- Return clear error messages for each failure point
