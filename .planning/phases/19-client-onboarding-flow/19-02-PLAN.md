**IMPORTANT:** This checkpoint requires a clean database state and real email for full verification.

**Pre-verification setup:**

1. Ensure Resend API key is configured (check .env or Vercel env vars)
2. Start dev server: `npm run dev`
3. Login as admin: http://localhost:3000/auth/login (team mode)
4. Navigate to Admin → Import Projects: http://localhost:3000/admin/projects/import

**Step 1: Configure project for portal**

1. Select a project from the list (checkbox)
2. Click "Configure Portal Settings" button
3. Fill in welcome message: "Welcome to your project portal! We're excited to work with you."
4. Enable all visibility toggles: Roadmap, Files, Comments
5. Click "Save Settings"
6. Verify status badge changes to "Portal Ready" (yellow)

**Step 2: Send invitation**

1. With same project selected, click "Send Invitation" button (green)
2. Enter your email address (use a real email you can access)
3. Click "Send Invitation"
4. Verify success toast appears
5. Verify status badge changes to "Invited" (blue)

**Step 3: Check invitation email**

1. Open your email inbox
2. Find email with subject: "You're invited to view {ProjectName} on Qualia"
3. Verify email contains:
   - Qualia teal gradient header
   - Project name
   - Welcome message you entered
   - "Create Account & View Project" button
4. Click the CTA button (or copy the link)

**Step 4: Signup flow**

1. Verify you land on signup page at /auth/signup?token=...
2. Verify page displays:
   - "Creating account for {ProjectName}" badge
   - Your email (pre-filled and read-only)
   - Welcome message in teal box
   - Full Name field (empty)
   - Password field (empty)
   - Confirm Password field (empty)
3. Fill in the form:
   - Full Name: "Test Client User"
   - Password: "testpass123"
   - Confirm Password: "testpass123"
4. Click "Create Account & Access Project" button
5. Verify loading spinner appears during submission

**Step 5: Verify auto-login and redirect**

1. Verify automatic redirect to /portal/{projectId} (no login screen)
2. Verify you see the project portal page with:
   - Project name in header
   - Sidebar navigation
   - Roadmap section visible
   - Phase cards displayed (if project has phases)

**Step 6: Verify full feature access**

1. **Roadmap access:**
   - Scroll to roadmap section
   - Verify phases are visible
   - Click on a phase to expand (if collapsible)
   - Verify no errors in browser console

2. **Files access:**
   - Navigate to Files section (sidebar or tab)
   - Verify "No files yet" message OR list of uploaded files
   - If files exist, click download icon
   - Verify file downloads successfully

3. **Comments access:**
   - Navigate to Comments section
   - Type a test comment: "Testing comment access after signup"
   - Click Submit
   - Verify comment appears in the list
   - Verify comment shows your name "Test Client User"

**Step 7: Verify database state**

Run these queries in Supabase SQL editor or via psql:

```sql
-- Check profile created with correct role
SELECT id, email, full_name, role, created_at
FROM profiles
WHERE email = 'your-email@example.com';
-- Expected: role = 'client'

-- Check client_projects link created
SELECT cp.*, p.name as project_name
FROM client_projects cp
JOIN profiles pr ON cp.client_id = pr.id
JOIN projects p ON cp.project_id = p.id
WHERE pr.email = 'your-email@example.com';
-- Expected: One row linking your user to the project

-- Check invitation status updated
SELECT email, status, opened_at, account_created_at, created_at
FROM client_invitations
WHERE email = 'your-email@example.com';
-- Expected: status = 'accepted', both timestamps populated
```

| Issue                                   | Likely Cause                    | Fix                                                |
| --------------------------------------- | ------------------------------- | -------------------------------------------------- |
| Email not received                      | Resend API key missing          | Check .env or Vercel env vars                      |
| Signup page shows "Invalid invitation"  | Token expired or wrong          | Use fresh invitation link                          |
| Signup fails with "User already exists" | Email already in auth.users     | Use different email or clear test data             |
| Redirect fails                          | Session not established         | Check browser console for auth errors              |
| Portal shows "Access denied"            | client_projects row not created | Check server action logs, verify adminClient works |
