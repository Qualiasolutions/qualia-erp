Create Supabase migration for notification_preferences table:

**Schema:**

- id (uuid, primary key, default gen_random_uuid())
- user_id (uuid, FK to profiles.id, unique)
- workspace_id (uuid, FK to workspaces.id)
- created_at (timestamptz, default now())
- updated_at (timestamptz, default now())

**Notification Type Toggles (boolean, default true):**

- notify_client_comment (employee notified when client comments)
- notify_client_file_upload (employee notified when client uploads file)
- notify_client_activity (employee notified on other client actions)
- notify_project_status_change (client notified when project status changes)
- notify_phase_milestone (client notified when phase completes/starts)

**Delivery Preferences (text, default 'both'):**

- delivery_method (enum: 'email', 'in_app', 'both')
  Create enum type: CREATE TYPE notification_delivery_method AS ENUM ('email', 'in_app', 'both')

**Indexes:**

- CREATE INDEX idx_notification_preferences_user_id ON notification_preferences(user_id)
- CREATE INDEX idx_notification_preferences_workspace_id ON notification_preferences(workspace_id)

**Constraints:**

- UNIQUE constraint on user_id (one preference row per user)
- FK constraints with ON DELETE CASCADE

**RLS Policies:**

- Enable RLS: ALTER TABLE notification_preferences ENABLE ROW LEVEL SECURITY
- SELECT policy: Users can view their own preferences
  CREATE POLICY "Users can view own preferences" ON notification_preferences FOR SELECT USING (auth.uid() = user_id)
- INSERT policy: Users can create their own preferences
  CREATE POLICY "Users can create own preferences" ON notification_preferences FOR INSERT WITH CHECK (auth.uid() = user_id)
- UPDATE policy: Users can update their own preferences
  CREATE POLICY "Users can update own preferences" ON notification_preferences FOR UPDATE USING (auth.uid() = user_id)
- DELETE policy: Users can delete their own preferences
  CREATE POLICY "Users can delete own preferences" ON notification_preferences FOR DELETE USING (auth.uid() = user_id)

After migration, regenerate TypeScript types:
npx supabase gen types typescript --local > types/database.types.ts

Create server actions file with 4 functions:

**1. getNotificationPreferences(userId: string): Promise&lt;NotificationPreferences | null&gt;**

- Query notification_preferences table by user_id
- Return null if no preferences exist (first-time user)
- Include workspace_id in response

**2. updateNotificationPreferences(formData: FormData): Promise&lt;ActionResult&gt;**

- Validate with Zod schema (notificationPreferencesSchema)
- Upsert preferences (INSERT ON CONFLICT(user_id) DO UPDATE)
- Set updated_at = now()
- Revalidate /settings and /settings/notifications paths
- Return ActionResult { success: true, data: preferences }

**3. createDefaultPreferences(userId: string, workspaceId: string): Promise&lt;ActionResult&gt;**

- Create preferences with all toggles = true, delivery_method = 'both'
- Called on first-time user accessing preferences page
- Return ActionResult { success: true, data: preferences }

**4. shouldSendEmail(userId: string, notificationType: string): Promise&lt;boolean&gt;**

- Helper function used by email notification functions
- Query user's preferences
- Return false if notification type is disabled
- Return false if delivery_method is 'in_app'
- Return true if no preferences exist (default: send emails)
- notification types: 'client_comment', 'client_file_upload', 'client_activity', 'project_status_change', 'phase_milestone'
