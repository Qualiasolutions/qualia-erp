1. Create Supabase migration `supabase/migrations/20260327120000_add_planned_logout_time.sql`:
   ```sql
   ALTER TABLE profiles ADD COLUMN IF NOT EXISTS planned_logout_time TIME DEFAULT NULL;
   COMMENT ON COLUMN profiles.planned_logout_time IS 'Employee planned end-of-shift time (e.g. 16:00). Used for clock-out reminders.';
   ```
   Apply via Supabase MCP or `supabase db push`. The TIME type stores HH:MM:SS without timezone ‚Äî perfect for a daily recurring schedule.

2. Update `types/database.ts` ‚Äî add `planned_logout_time: string | null` to the profiles Row, Insert (optional), and Update (optional) type definitions. The field stores a TIME as string like "16:00:00".

3. Add to `app/actions/work-sessions.ts`:
   - `getPlannedLogoutTime(workspaceId: string): Promise<string | null>` ‚Äî fetches current user's `planned_logout_time` from profiles. Returns the TIME string or null.
   - `updatePlannedLogoutTime(workspaceId: string, time: string | null): Promise<ActionResult>` ‚Äî updates current user's `planned_logout_time`. Validate format with regex `/^\d{2}:\d{2}(:\d{2})?$/` before saving. If `time` is null or empty string, set to null (clear it). Calls `revalidatePath('/settings')`.

4. Add SWR hook to `lib/swr.ts`:
   - Add cache key: `plannedLogout: (workspaceId: string) => ['planned-logout', workspaceId] as const`
   - Add `usePlannedLogoutTime(workspaceId: string | null)` ‚Äî fetches via `getPlannedLogoutTime`. Refresh interval: 0 (no auto-refresh, only on mount). Returns `{ plannedLogoutTime: string | null, isLoading: boolean }`.
   - Add `invalidatePlannedLogoutTime(workspaceId: string, immediate?: boolean)`.
     </action>

1. Create `hooks/use-beforeunload-guard.ts`:
   - `'use client'` directive.
   - Export `useBeforeunloadGuard(enabled: boolean)`.
   - When `enabled` is true, add `beforeunload` event listener that calls `e.preventDefault()` (the standard way to trigger native browser confirmation).
   - Clean up listener when `enabled` becomes false or on unmount.
   - Simple hook ‚Äî no UI, just the browser-native "Leave site?" dialog.

2. Create `components/planned-logout-banner.tsx`:
   - `'use client'` component.
   - Uses `useActiveSession`, `useCurrentWorkspaceId`, `usePlannedLogoutTime`, `useAdminContext`.
   - Only renders for employees with active sessions AND a `planned_logout_time` set.
   - Logic: Parse `planned_logout_time` (e.g., "16:00:00"), compare with current local time. If current time > planned logout, show banner. Use a 1-minute interval to re-check.
   - Banner design: A slim, non-intrusive bar at the top of the content area. Use `AlertTriangle` icon (lucide). Amber/warning color scheme (not red ‚Äî it's a reminder, not an error). Text: "Your shift ended at {time}. Don't forget to clock out!" with a small "Dismiss" button on the right.
   - Dismissible via local state (`dismissed`). Reset `dismissed` to false on route change (use `usePathname` ‚Äî when pathname changes, un-dismiss). This makes it persistent across navigation but dismissible on the current page.
   - Also uses `useBeforeunloadGuard(hasActiveSession)` ‚Äî active whenever there's an active session, regardless of planned logout time.
   - Smooth entrance animation: translate-y from -full to 0, opacity 0 to 1, duration-300 ease-out-quart. Use Tailwind classes with conditional rendering.

3. Update `app/settings/page.tsx`:
   - Add a "Work Schedule" section to the settings page.
   - Show a time input field for "Planned logout time" with a label like "When does your shift end?".
   - Use a native `<input type="time">` styled with Tailwind (simpler and more accessible than a custom time picker). Wrap in shadcn Card for consistency.
   - On change: call `updatePlannedLogoutTime` server action, then `invalidatePlannedLogoutTime`. Show success toast via sonner.
   - Pre-populate from `usePlannedLogoutTime` hook.
   - Include a "Clear" button to remove the time (set to null).

4. Add `<PlannedLogoutBanner />` to `app/layout.tsx`:

   - Wrap in `<Suspense fallback={null}>`.
   - The banner sits between the sidebar edge and the main content scroll area.
