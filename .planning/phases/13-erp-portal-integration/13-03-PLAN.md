3. Server component renders client wrapper with projectId

Why this pattern: Server component handles security (auth + access) which cannot be bypassed. Client component handles data fetching with auto-refresh which updates without full page reload.

Implementation:

- Create PortalProjectContent component in same file or separate file (your choice)
- Use 'use client' directive
- Import and use usePortalProjectWithPhases
- Pass live data to PortalRoadmap (phases, project, isLoading, isValidating)
- Maintain existing PortalPageHeader and PortalTabs components
- Keep userRole detection (admin/employee/client logic)

Note: User role detection can stay in server component OR move to client component. If moved to client, fetch profile via Supabase client.
