<task type="auto">
  <name>Task 3: Add prefers-reduced-motion support and dark mode transition styles</name>
  <files>app/globals.css</files>
  <action>
Add two CSS rules at the end of globals.css:

1. **Dark mode smooth transitions:** Add transition rule for all elements when theme changes
2. **Reduced motion support:** Disable all animations for users who prefer reduced motion

Add this block at the end of the file (after all other rules):

```css
/* Smooth dark mode transitions */
* {
  transition-property: color, background-color, border-color, text-decoration-color, fill, stroke;
  transition-timing-function: cubic-bezier(0.16, 1, 0.3, 1);
  transition-duration: 300ms;
}

/* Respect prefers-reduced-motion (A11Y-01) */
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

**Why these rules work:**

Dark mode transitions:

- Scoped to color-related properties (not layout properties like width/height)
- Uses premium easing from tailwind.config.ts (0.16, 1, 0.3, 1)
- 300ms duration matches theme toggle expectation
- Only triggers when theme class changes on html element

Reduced motion:

- Overrides ALL animations and transitions to near-instant (0.01ms)
- Catches Tailwind animations, Framer Motion, and custom CSS animations
- Includes ::before and ::after pseudo-elements
- Disables smooth scroll behavior (can trigger motion sickness)
- Uses !important to ensure it wins over all animation declarations

**Testing:** User can test by enabling "Reduce motion" in OS accessibility settings. All page transitions, stagger animations, and theme transitions will become instant.

**Coverage:** This makes TRANS-01, TRANS-02, TRANS-03, DARK-01, and A11Y-01 all work together harmoniously.
