  <action>
Add subtle spring physics to button press interactions using Framer Motion.

**Approach:**

1. Import motion: `import { motion } from 'framer-motion';`
2. Create a motion variant of the Button component using forwardRef pattern
3. Wrap button element with motion.button or create MotionButton primitive
4. Add spring-based tap animation:
   ```tsx
   <motion.button
     whileTap={{
       scale: 0.97,
       transition: {
         type: 'spring',
         stiffness: 400,
         damping: 17,
       },
     }}
     {...props}
   >
     {children}
   </motion.button>
   ```

**Spring configuration rationale:**

- `stiffness: 400` - Responsive but not snappy (not 500+)
- `damping: 17` - Smooth settle, minimal bounce (not elastic like 10)
- `scale: 0.97` - Subtle press (not dramatic 0.95)

**Important considerations:**

- Preserve existing Button variants (default, destructive, outline, ghost, link, secondary)
- Ensure className merging still works (cn utility)
- Preserve asChild prop for Slot composition
- Spring should only apply to actual button interactions (not disabled buttons)
- Check that existing click handlers aren't blocked by motion wrapper

**Testing pattern:**
Test with all button variants to ensure spring works consistently across default, outline, ghost, destructive, etc.

**Reduced motion support:**

Framer Motion v12+ automatically respects `prefers-reduced-motion` media query via built-in support. Additionally, Phase 5 added CSS override in globals.css that forces transition-duration to 0.01ms for users with reduced motion enabled. Both mechanisms ensure spring animations degrade gracefully to instant state changes (no spring bounce).
