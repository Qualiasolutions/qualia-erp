**Fix 1 — `next.config.ts` removeConsole (line ~97):**

Read `next.config.ts` to see the full current config. Change the `removeConsole` option from:

```ts
removeConsole: process.env.NODE_ENV === 'production',
```

to:

```ts
removeConsole:
  process.env.NODE_ENV === 'production'
    ? { exclude: ['error', 'warn'] }
    : false,
```

This uses Next.js's `removeConsole.exclude` option to preserve `console.error` and `console.warn` while still stripping `console.log`, `console.info`, `console.debug` in production.

**Fix 2 — Health endpoint 503 for degraded (`app/api/health/route.ts`, line ~83):**

Change line 83 from:

```ts
const statusCode = health.status === 'healthy' ? 200 : health.status === 'degraded' ? 200 : 503;
```

to:

```ts
const statusCode = health.status === 'healthy' ? 200 : 503;
```

Both `degraded` and `unhealthy` now return 503, which is the correct HTTP semantics for a service that is not fully operational.
