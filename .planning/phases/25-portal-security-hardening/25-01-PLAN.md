**Fix 1 — `updateFeatureRequest` IDOR (around line 145):**

After getting `user` from `supabase.auth.getUser()`, check if the user is an admin/manager:


Then add `.eq('client_id', user.id)` to the update query when the user is NOT an admin:

```ts
let query = supabase.from('client_feature_requests').update(updateData).eq('id', requestId);

if (!isAdmin) {
  query = query.eq('client_id', user.id);
```

**Fix 2 — `createFeatureRequest` project_id ownership check (around line 28):**

After validating title and before the insert, if `input.project_id` is provided, verify it belongs to the client:

```ts
if (input.project_id) {

    .from('client_projects')
    .select('project_id')
    .eq('client_id', user.id)
    .eq('project_id', input.project_id)
    .single();

    return { success: false, error: 'Project not found or access denied' };
  }
}
```

This check only runs when `project_id` is supplied (it is optional). When no project_id is given, the insert proceeds as before.

**Fix 3 — Phase comments IDOR (`app/actions/phase-comments.ts`):**

Add a helper function `canAccessProject` at the top of the file (after imports):

```ts
async function canAccessProject(userId: string, projectId: string): Promise<boolean> {
  const supabase = await createClient();

  // Admins and employees can access all projects
  const { data: profile } = await supabase
    .from('profiles')
    .select('role')
    .eq('id', userId)
    .single();

  if (profile?.role === 'admin' || profile?.role === 'employee' || profile?.role === 'manager') {
    return true;
  }

  // Clients must be linked via client_projects
  const { data: link } = await supabase
    .from('client_projects')
    .select('project_id')
    .eq('client_id', userId)
    .eq('project_id', projectId)
    .single();

  return !!link;
}
```

```ts
if (!(await canAccessProject(user.id, projectId))) {
```
