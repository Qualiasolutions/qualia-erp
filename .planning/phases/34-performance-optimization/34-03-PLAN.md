  <action>
framer-motion provides a `LazyMotion` component + `m` namespace that enables tree-shaking and lazy feature loading. This is the official way to reduce bundle size.

**Step 1: Create `lib/lazy-motion.tsx`**

```tsx
'use client';

import { LazyMotion, domAnimation } from 'framer-motion';
import type { ReactNode } from 'react';

export function LazyMotionProvider({ children }: { children: ReactNode }) {
  return (
    <LazyMotion features={domAnimation} strict>
      {children}
    </LazyMotion>
  );
}

// Re-export m for convenience
export { m } from 'framer-motion';
export { AnimatePresence } from 'framer-motion';
```

Note: We use `domAnimation` (not `domMax`) because the codebase only uses basic features (animate, exit, variants, gestures). `domMax` adds layout animations which aren't used. `domAnimation` is ~17KB vs ~32KB for the full bundle.

**Step 2: Add LazyMotionProvider to the root layout**

Find the root `app/layout.tsx` and wrap the children (or the body content) with `<LazyMotionProvider>`. This ensures ALL child components can use `m` without their own provider.

```tsx
import { LazyMotionProvider } from '@/lib/lazy-motion';
// ... in the JSX:
<LazyMotionProvider>{children}</LazyMotionProvider>;
```

**Step 3: Update all 20 component files**

For EACH file that imports from `'framer-motion'`:

1. Replace `import { motion, AnimatePresence } from 'framer-motion'` with `import { m, AnimatePresence } from '@/lib/lazy-motion'`
2. Replace `import { motion } from 'framer-motion'` with `import { m } from '@/lib/lazy-motion'`
3. Replace `import { motion, useInView } from 'framer-motion'` with `import { m } from '@/lib/lazy-motion'` + `import { useInView } from 'framer-motion'` (hooks still from framer-motion directly)

6. Replace ALL `<motion.` with `<m.` and `</motion.` with `</m.`
