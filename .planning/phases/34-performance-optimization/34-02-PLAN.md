**Part A: Create the PostgreSQL RPC `batch_update_task_orders`**

Create the migration file `supabase/migrations/20260327_batch_update_task_orders.sql`:

```sql
CREATE OR REPLACE FUNCTION batch_update_task_orders(updates jsonb)
RETURNS void
LANGUAGE plpgsql
SECURITY DEFINER
AS $$
DECLARE
  item jsonb;
BEGIN
  FOR item IN SELECT * FROM jsonb_array_elements(updates)
  LOOP
    UPDATE tasks
    SET
      sort_order = (item->>'sort_order')::int,
      status = COALESCE(item->>'status', status),
      completed_at = CASE
        WHEN item->>'status' = 'Done' THEN now()
        WHEN item ? 'status' AND item->>'status' != 'Done' THEN NULL
        ELSE completed_at
      END
    WHERE id = (item->>'id')::uuid;
  END LOOP;
END;
$$;
```

This processes ALL task updates in a single DB round-trip. The function:

- Accepts a JSONB array of `{id, sort_order, status?}` objects
- Updates each task's sort_order, and optionally status + completed_at
- Uses `SECURITY DEFINER` so it runs with the function owner's privileges (safe because callers are already authenticated server actions)

Apply the migration via Supabase MCP or CLI: `supabase db push` or apply through the Supabase dashboard SQL editor.

**Part B: Wrap auth helpers with React `cache()` in shared.ts**

Import `cache` from `'react'` at the top of `app/actions/shared.ts`.

Create a cached internal function for getting user role:

```typescript
import { cache } from 'react';

const getCachedUserRole = cache(async (userId: string): Promise<string | null> => {
  const supabase = await createClient();
  const { data } = await supabase.from('profiles').select('role').eq('id', userId).single();
  return data?.role || null;
});
```

Rewrite `isUserAdmin`, `isUserManagerOrAbove`, and `getUserRole` to use `getCachedUserRole`:

```typescript
export async function isUserAdmin(userId: string): Promise<boolean> {
  const role = await getCachedUserRole(userId);
  return role === 'admin';
}

export async function isUserManagerOrAbove(userId: string): Promise<boolean> {
  const role = await getCachedUserRole(userId);
  return role === 'admin' || role === 'manager';
}

export async function getUserRole(userId: string): Promise<string | null> {
  return getCachedUserRole(userId);
}
```

This means within a single request, calling `isUserAdmin(id)` + `isUserManagerOrAbove(id)` + `canDeleteMeeting(id, meetingId)` only queries the DB ONCE for the role.

**Part C: Optimize reorderTasks in inbox.ts to use the batch RPC**

Current problem: For non-admin users, `reorderTasks` calls `canModifyTask` in a sequential loop (N queries). Then it does N individual UPDATE calls via `Promise.all`.

New approach — 2 total DB round-trips:

**Round-trip 1: Batch auth check** (for non-admin users only). Replace the sequential `canModifyTask` loop (lines 513-522) with a single batch query:

```typescript
if (!isAdmin) {
  // Batch auth check: fetch all tasks in one query, verify ownership
  const taskIds = taskUpdates.map(({ id }) => id);
  const { data: tasks } = await supabase
    .from('tasks')
    .select('id, creator_id, assignee_id, project:projects(lead_id)')
    .in('id', taskIds);

  if (!tasks || tasks.length !== taskIds.length) {
    return { success: false, error: 'One or more tasks not found' };
  }

  for (const task of tasks) {
    const isCreator = task.creator_id === user.id;
    const isAssignee = task.assignee_id === user.id;
    const project = Array.isArray(task.project) ? task.project[0] : task.project;
    const isLead = project?.lead_id === user.id;
    if (!isCreator && !isAssignee && !isLead) {
      return { success: false, error: 'You do not have permission to reorder one or more tasks' };
    }
  }
}
```

**Round-trip 2: Batch update via RPC.** Replace the entire `Promise.all` block (lines 525-550) with a single RPC call:

```typescript
// Build the updates JSONB array
const updates = taskUpdates.map(({ id, sort_order, status }) => {
  const update: { id: string; sort_order: number; status?: string } = { id, sort_order };
  if (status !== undefined) {
    update.status = status;
  }
  return update;
});

const { error } = await supabase.rpc('batch_update_task_orders', {
  updates: JSON.stringify(updates),
});

if (error) {
  console.error('[reorderTasks] Batch update error:', error);
  return { success: false, error: 'Failed to reorder tasks' };
}

revalidatePath('/inbox');
return { success: true };
```

This achieves the target: 1 auth check query + 1 RPC batch update = 2 DB round-trips max (admins skip the auth check = 1 round-trip).
