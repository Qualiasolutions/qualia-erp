**Part A: Wrap auth helpers with React `cache()` in shared.ts**

Import `cache` from `'react'` at the top of `app/actions/shared.ts`.

Create a cached internal function for getting user role:

```typescript
import { cache } from 'react';

const getCachedUserRole = cache(async (userId: string): Promise<string | null> => {
  const supabase = await createClient();
  const { data } = await supabase.from('profiles').select('role').eq('id', userId).single();
  return data?.role || null;
});
```

Rewrite `isUserAdmin`, `isUserManagerOrAbove`, and `getUserRole` to use `getCachedUserRole`:

```typescript
export async function isUserAdmin(userId: string): Promise<boolean> {
  const role = await getCachedUserRole(userId);
  return role === 'admin';
}

export async function isUserManagerOrAbove(userId: string): Promise<boolean> {
  const role = await getCachedUserRole(userId);
  return role === 'admin' || role === 'manager';
}

export async function getUserRole(userId: string): Promise<string | null> {
  return getCachedUserRole(userId);
}
```

This means within a single request, calling `isUserAdmin(id)` + `isUserManagerOrAbove(id)` + `canDeleteMeeting(id, meetingId)` only queries the DB ONCE for the role.

**Part B: Optimize reorderTasks auth check in inbox.ts**

Current problem: For non-admin users, `reorderTasks` calls `canModifyTask` in a sequential loop — each call creates a new Supabase client and queries the DB.

New approach: Replace the sequential loop with a single batch query.

Replace lines 513-522 in inbox.ts with:

```typescript
if (!isAdmin) {
  // Batch auth check: fetch all tasks in one query, verify ownership
  const taskIds = taskUpdates.map(({ id }) => id);
  const { data: tasks } = await supabase
    .from('tasks')
    .select('id, creator_id, assignee_id, project:projects(lead_id)')
    .in('id', taskIds);

  if (!tasks || tasks.length !== taskIds.length) {
    return { success: false, error: 'One or more tasks not found' };
  }

  for (const task of tasks) {
    const isCreator = task.creator_id === user.id;
    const isAssignee = task.assignee_id === user.id;
    const project = Array.isArray(task.project) ? task.project[0] : task.project;
    const isLead = project?.lead_id === user.id;
    if (!isCreator && !isAssignee && !isLead) {
      return { success: false, error: 'You do not have permission to reorder one or more tasks' };
    }
  }
}
```

This replaces N sequential queries with 1 batch query.

Also, the individual `supabase.from('tasks').update()` calls in the Promise.all (lines 525-541) are already parallelized which is good. Leave those as-is — they can't easily be batched into a single RPC without creating a stored procedure, and the parallel updates are adequate.
