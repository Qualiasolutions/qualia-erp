System Connections

How GitHub, Claude Code, and the Qualia Portal synchronize project state automatically.

Data Flow: Code to Portal

Developer pushes code to GitHub
GitHub Webhook fires (push event)
Portal API receives at /api/github/webhook
Parses .planning/phases/ files from repo
Creates/updates project_phases in Supabase
Staff sees progress in Portal — Client sees progress in Client Portal

Auto-Update Triggers

Phase status is automatically derived from file changes in the .planning/ directory.

PLAN.md created → in_progress
SUMMARY.md created → in_progress
VERIFICATION.md created → completed
CONTEXT.md updated → in_progress

The project_integrations Table

Links Supabase projects to their GitHub repos. Stores webhook secrets, repo names, and last sync timestamps.

Claude Code Integration

Claude Code runs locally and pushes to GitHub, which triggers the webhook pipeline. The GSD system generates .planning/ files as it works through phases, giving the portal real-time visibility.

# Claude Code workflow 1. Claude Code receives task 2. GSD system creates .planning/phases/phase-N/ 3. Writes PLAN.md, executes tasks, writes SUMMARY.md 4. Runs verification, writes VERIFICATION.md 5. Commits and pushes to GitHub 6. Webhook fires → Portal updates automatically

Phase Review Workflow

When a phase is completed, it enters the review pipeline. Staff submits for review, and the client can approve or request changes.

Phase Complete → Submit for Review → Client Reviews → Approved

If changes are requested, the phase returns to active status with client feedback attached.

Phase Status States

StatusBadgeMeaning

in_progressIn ProgressActive development work
