03 · Framework commands

The handful you’ll use every day.

The framework has many commands. You don’t need to memorise all of them. Memorise these six. For everything else, type /qualia-help.

/qualia

The smart router. Don’t know what’s next? Type this. It tells you the exact command for your situation.

/qualia-new

Start a brand new project. Asks you the right questions, runs research, builds the journey from scratch.

/qualia-plan

Plan the current phase. The framework spawns a planner, validates the plan, gives you something concrete to build.

/qualia-build

Execute the current phase. Wave-based subagents run in parallel. You watch, you intervene if needed.

/qualia-report

Mandatory at end of every session. Generates the report and posts it to the ERP. No report = session not counted.

/qualia-ship

Deploy to production. Full quality gates, then vercel --prod. Use this when a project is genuinely done.

04 · What changed

Before April · After May.

If you’re used to the old way, this is what’s different now. None of this is criticism — it’s the system levelling up.

Before · April

  • 422 inbox tasks auto-generated from .planning/ checklists
  • Clock-in optional — many sessions never recorded
  • Report submitted “when remembered”, often missed
  • Project assignments without deadlines
  • No clear unit of accountability
  • Admin had no single page to see “what needs attention today”

After · May

  • Tasks archived. Project is the unit. We agree on May projects together
  • Mandatory clock-in & clock-out, every working day
  • /qualia-report is part of the clock-out flow, not optional
  • Every project has a deadline you helped set
  • You own the project end-to-end. Deploy URL is the proof
  • The new Command Center shows everyone’s status in one glance
