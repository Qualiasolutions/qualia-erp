The big idea, in one sentence.

v5 makes the framework remember what your project means and prevent the dumb mistakes that wasted hours before.

When you start a project, Qualia now writes a small glossary of your domain words. Every agent reads it before doing anything, so nobody ever again confuses a “user” for a “customer”, or builds an “Ophthalmos” page when the client is “Ophthemous.” Decisions get written down once, honored everywhere.

On top of that, three new safety hooks block the most painful mistakes deterministically — wrong Vercel team, empty env vars on prod, accidental database wipes. You don’t have to remember; the framework remembers for you.

Misalignment is the #1 reason AI work goes sideways. v5 fixes it at the root.
01 · What’s new

Five commands you didn’t have before.

Each one solves a specific friction. You don’t need all of them every day, but knowing they exist saves you from reinventing the workaround.

Reference
/qualia-zoom

I don’t know this code

Staring at a file you’ve never seen? Type /qualia-zoom and the agent gives you a 30-second map: what these files do, who calls them, what the safe edit zone is. Read-only. User-fired only — never auto.

Use when: picking up a project mid-flight, before editing unfamiliar code.
Workflow
/qualia-issues

Externalize the plan

Take a phase plan and turn it into independent vertical-slice GitHub issues. Each one demoable on its own, dependency-ordered. Now another teammate, another session, or an autonomous agent can pull from the queue.

Use when: the work needs more than one person, or you want to hand off to autonomous build.
Workflow
/qualia-triage

Route the queue

Walks every open issue and labels it: ready-for-agent, ready-for-human, needs-info, or wontfix. Optionally pulls ready-for-agent issues into autonomous /qualia-build. Pairs with /qualia-issues.

Use when: the issue queue is overflowing or you want autonomous building from the backlog.
Reference
/qualia-road

Built-in cheat sheet

The full workflow map, every command, when to use each one — in your terminal. No need to open the docs. Drier than a tutorial; quicker than asking.

Use when: you forget which command does what. Type it, scan, move on.
Experimental · flagship
/qualia-polish-loop {url}

Claude finally sees the page.

⚠ Test on a non-critical page before relying on it for a client deploy. The biggest thing in v5. Until now, design quality was scored by reading your CSS. v5 introduces a loop that actually screenshots your page at three viewports (mobile, tablet, desktop), evaluates against the rubric using vision, fixes the top issues, redeploys to a preview, and repeats — until every dimension scores 3 or higher, or a kill-switch stops the loop. Per-iteration commits with qpl-N: prefix so any iteration is revertable.

Use when: hero videos broken on mobile, layout looks off and you can’t tell why, design review needed before ship.
02 · Upgraded

Five commands that got smarter.

Same names, same triggers. The behavior underneath levels up.

/qualia-discuss
Phase context capture

Before

Open-ended interview. Asked broad questions. You did most of the work figuring out what to lock in.

Now

Relentless one-question-at-a-time grilling. Proposes a recommended answer for each. Walks every branch. Updates CONTEXT.md inline as terms get resolved.

/qualia-optimize
Codebase quality sweep

Before

Six dimensions: perf, design, UI, backend, frontend, alignment. Found surface-level issues.

Now

Adds --deepen mode. Ousterhout-style architecture audit. Finds shallow modules. Refuses “extract a helper” candidates that don’t pass the deletion test.

/qualia-test
Test generation + run

Before

Generated tests after the code was written. Ran tests. Reported coverage.

Now

Adds --tdd mode. Drives a feature test-first via vertical-slice red-green-refactor. One test, one minimal impl, repeat. Bans horizontal slicing.

/qualia-map
Brownfield codebase mapping

Before

Four parallel agents scanned architecture, stack, conventions, concerns.

Now

Five parallel agents. The fifth detects existing GitHub/GitLab issue tracker, label conventions, glossary location, existing CLAUDE.md. Adapts Qualia to the repo’s process instead of overwriting.

/qualia-report
End-of-session clock-out

Before

Forced a report even on empty days. Vague synthesis template. Could leave you stuck on errors with no fix path.

Now

Empty-day handling — asks gracefully if you didn’t commit. Structured synthesis template. Encouraging closing message. Self-service common-errors table. Full ERP integration preserved.

03 · Safety nets

Three new hooks. The framework remembers so you don’t have to.

Every one of these is something Fawzi has been bitten by. v5 makes them impossible to do by accident.

Pre-deploy

Wrong Vercel team? Blocked.

Before vercel --prod runs, the guard checks vercel whoami against your allowed teams list. If it doesn’t match, the deploy stops before it starts.

$ vercel --prod ✗ BLOCKED: Wrong Vercel team. Active: 'random-team'. Run vercel switch <team> first.
Pre-env-write

Empty env var on prod? Blocked.

Detects vercel env add NAME "" patterns before they ship. Empty string on production has crashed real apps. Bypass with QUALIA_ALLOW_EMPTY_ENV=1 if intentional.

$ vercel env add API_KEY "" ✗ BLOCKED: empty env var value detected.
Pre-supabase

Database wipe? Blocked.

Stops supabase db reset, db push --force, and migration repair from running by accident. Bypass with QUALIA_ALLOW_DESTRUCTIVE=1 when sure.

$ supabase db reset ✗ BLOCKED: destructive Supabase command.
Plus two smaller guards: Suspicious project names (test, tmp, scratch) need --force to create. And QUALIA_SKIP_LINT=1 lets you ship past pre-existing lint errors in an emergency, while keeping all other quality gates active.
