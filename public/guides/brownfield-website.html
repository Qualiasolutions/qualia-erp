Existing Website

Work on Existing Website

Add features or fix bugs

1 Go to the Project Folder
cd ~/Desktop/Projects/websites/existing-project-name
2 Check Git Status
git status && git branch

Make sure you're on the right branch and have no uncommitted changes.

3 Start Claude Code
claude
4 Explore the Codebase First

ALWAYS do this before making changes!

/brownfield-explore

Tell Claude what you need to understand:

"I need to add a new pricing page. Show me how pages are structured in this project and where the pricing data might come from."
5 Create a Feature Branch
git checkout -b feature/pricing-page
6a For Small Changes - Use Quick Mode
/gsd:quick

Tell Claude what to do:

"Add a new pricing page at /pricing with 3 tiers: Basic ($9/mo), Pro ($29/mo), Enterprise (Contact us). Match the existing page style and component patterns."
6b For Bigger Features - Use Full GSD
/gsd:new-milestone

Then work through phases:

/gsd:discuss-phase 1 /gsd:plan-phase 1 /gsd:execute-phase 1
7 If You Need UI Design Help
/fd
"Design a pricing cards section that matches the existing site style. 3 cards with features lists and CTA buttons."
8 If Something Breaks
/sf
"The pricing page throws an error when I click the Pro plan button. Here's the error: [paste error]"
9 Test Your Changes

Run the app locally and check:

  • Your new feature works
  • Old features still work
  • Mobile looks good
10 Save & Create PR
/git-flow

Tell Claude:

"Commit my changes and create a pull request"
11 Deploy
/quick-deploy

Before You Deploy - Check These

  • Explored codebase first
  • Followed existing patterns
  • New feature works
  • Old features still work
  • Mobile responsive
  • No console errors
