Existing Voice

Modify Existing Voice Agent

Change VAPI agent behavior or add features

1 Go to the Project Folder
cd ~/Desktop/Projects/voice/existing-voice-agent
2 Check Git Status
git status && git branch
3 Start Claude Code
claude
4 Explore the Webhook Code
/brownfield-explore
"Show me how this voice agent works. What VAPI webhooks does it handle? What functions does the agent have?"
5 Check VAPI Dashboard

Go to VAPI dashboard and note:

  • Assistant name and ID
  • Current system prompt
  • Functions defined
  • Voice settings
6 Create Feature Branch
git checkout -b feature/add-transfer
7 Make Your Changes

To change what the agent says:

Edit the system prompt in VAPI Dashboard → Assistant → Edit

To add a new function:

/gsd:quick
"Add a transfer_to_human function to the VAPI agent. When called, it should transfer the call to +1234567890. Add the handler in the webhook."

To change the voice:

VAPI Dashboard → Assistant → Voice → Choose new voice

To add a new webhook handler:

/supabase-edge
"Add a handler for the 'transcript' event that saves each message in realtime to Supabase."
8 Test in VAPI Dashboard First

VAPI Dashboard → Your Assistant → Test tab → Make a web call

9 Test with Real Phone Call

Call your number and test:

  • New feature works
  • Old features still work
  • No awkward pauses
  • Functions respond quickly
10 If Something Doesn't Work
/sf
"The transfer_to_human function isn't working. The agent says it will transfer but nothing happens. Here's what I see in the logs: [paste logs]"
11 Deploy Webhook Changes
/git-flow /quick-deploy

Note: VAPI dashboard changes are instant. Only webhook code needs deploying.

Before You Go Live - Check These

  • Web call test passed
  • Real phone call test passed
  • All functions work
  • No awkward silences
  • Webhook URL is correct
  • Calls save to database
