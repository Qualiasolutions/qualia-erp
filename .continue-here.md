## Decisions made

- Kept the `escapeHtml` OWASP pattern as a formalized tested lib rather than swapping to DOMPurify (would add ~20KB client bundle for 5 lines of logic). Unit tests cover XSS payloads, order-of-escape invariant, edge cases.
- Made the `session_reports_client_report_id_uniq` index **plain** (not partial) because Supabase's `.upsert({...}, { onConflict })` doesn't pass the partial WHERE clause to the resolver. Composite-UNIQUE NULL semantics mean legacy rows with NULL client_report_id still coexist fine.
- Response shape for `POST /api/v1/reports` is **conditional**: v4.0.4 payloads get `report_id: "QS-REPORT-NN"`; legacy payloads get `report_id: <UUID>` (unchanged behavior). Backward compat preserved.
