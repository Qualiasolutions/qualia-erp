## Decisions Made

- **UI status mapping over DB enum migration on requests kanban.** Kept DB enum unchanged (still has `in_review`, `planned`, `declined`); the UI maps `in_review` → Pending, `planned` → In Progress, `declined` → Archived (off-board). Drag writes only canonical statuses (pending/in_progress/completed) — never the legacy values. Avoids the CHECK-constraint silent-fail trap from the Sakani lesson.
- **`/api/claude/*` deletion not just deprecation.** The framework migrated to `/api/v1/reports` per its own CHANGELOG; no internal callers. Deleting the routes closes both dead-code AND auth-bypass at once.
- **`revalidate = 60` removed from /status and /knowledge** (caught by adversarial verifier — incompatible with `nextConfig.cacheComponents: true`). Caching for these segments should use `'use cache'` directive, not segment exports.
- **React Compiler rules at `warn`, not `error`.** B1 surfaced 72 violations; fixing them all in-session would have blown context budget. Warn level catches new code, leaves existing surface for follow-up.
- **`lib/swr.ts` barrel preserves backward-compat.** Splitting into `lib/swr/*.ts` but keeping `@/lib/swr` import path for all 34 consumers. Future migration to direct imports is non-blocking.
- **lint-staged stash bug now visible.** The empty-commit detector in `.husky/post-commit` catches it within seconds. Hit this twice today (Phase 28's `dfbf13d2` came out empty; re-committed as `e39970a3`). With the post-commit hook in place, future occurrences will warn loudly.
